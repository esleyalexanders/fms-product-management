Manage Team Module

Business Requirements Document

Version 1.0 Ready for Development

Feature Requirements

1. Staff Management (CRUD)

1.1 Create Staff

User Story:
As a Manager, I want to add new staff members with their personal information, employment details, and scheduling preferences so that they can be assigned to jobs and shifts.
Requirements:
  • Multi-tab form interface (Personal Info, Employment & Scheduling)
  • Profile photo upload (optional, max 5MB, JPG/PNG)
  • Real-time form validation
  • Option to send invitation email to staff member
  • Auto-save functionality (future enhancement)

1.2 View Staff

User Story:
As a Manager, I want to view a list of all active staff with search and filter capabilities so that I can quickly find and manage team members.
Features:
  • Table view with key information (Name, Role, Email, Phone, Status)
  • Real-time search by name, email, phone
  • Filter by role and status (active/inactive)
  • Pagination (if list exceeds 50 items)
  • Export to CSV functionality
  • Detailed staff profile view

1.3 Edit Staff

User Story:
As a Manager, I want to update staff information, roles, and scheduling preferences so that records stay current and accurate.
Features:
  • Pre-populated form with existing data
  • Same validation rules as create
  • Option to send update notification
  • Audit trail of changes (backend)

1.4 Deactivate/Reactivate Staff

User Story:
As a Manager, I want to deactivate staff accounts when employees leave while maintaining historical data for compliance and reporting.
Features:
  • Soft delete (status change, not physical deletion)
  • Confirmation dialog before deactivation
  • Prevent deactivated staff from being assigned to new jobs
  • Ability to reactivate staff
  • Visual distinction (red for deactivate, green for reactivate)

2. Role and Permission Management

2.1 Predefined Roles

Role Description Key Permissions
Manager Full system access All features, manage roles, system settings, manage staff
Staff General team members (technicians, receptionists, field workers) View/update assigned jobs, view schedule, manage bookings, view customers
Important: Staff members have both a Role (determines permissions) and an optional Job Title (display name). Example: Role = "Staff", Job Title = "Senior HVAC Technician" or "Lead Receptionist"

2.2 Permission Matrix

Six permission groups control access:

  1. Staff Management: View, Create, Edit, Delete staff
  2. Schedule Management: View, Edit schedules, Assign jobs
  3. Quote Management: View, Create, Edit quotes
  4. Customer Management: View, Create, Edit customers
  5. Reports & Analytics: View reports, Export data
  6. System Administration: Manage roles, System settings

3. Scheduling & Availability

Weekly Availability

  • Per-day time ranges
  • Multiple ranges per day
  • Weekday preset (9-5 Mon-Fri)

Time Off

  • Blocked date ranges
  • Vacation/leave tracking
  • Multiple blocked periods

Work Constraints

  • Max hours per week
  • Max consecutive days
  • Min shift length
  • Shift padding/buffer

Home Base

  • Use store address (default)
  • Custom address option
  • For route planning

Locations & Skills

  • Assigned locations
  • Skills/specializations
  • Comma-separated values

User Workflows

Staff Creation Workflow

1
Navigate to Team List

User clicks "Manage Team" in sidebar → Sees staff list page

2
Click "Add Staff" Button

User clicks primary action button → Navigates to create form

3
Fill Personal Information Tab

Required: First Name, Last Name, Email, Phone
Optional: Profile Photo, Date of Birth

4
Switch to Employment & Scheduling Tab

User clicks tab → Sees employment and scheduling sections

5
Fill Employment Information

Required: Start Date, Employment Type, Role
Optional: Employee ID, Hourly Rate, Job Title

6
Configure Scheduling (Optional)

Set weekly availability, time off, work constraints, home base

7
Submit Form

Validation runs → Success: Save & redirect to list → Error: Show inline errors

8
Send Invitation (Optional)

If checkbox checked → System sends invitation email to staff member

Success Criteria: Staff member is created, appears in list, can be assigned to schedules and jobs

Staff Edit Workflow

1
Navigate to Edit

From list: Click "Edit" button → Or from detail view: Click "Edit Staff"

2
Form Pre-populated

System loads existing staff data → All fields filled with current values

3
Make Changes

User modifies any fields → Can deactivate staff via button at top

4
Save Updates

Validation runs → Success: Save & redirect → Notification sent if checkbox checked

Search & Filter Workflow

1
Enter Search Term

User types in search box → Real-time filtering by name, email, phone

2
Apply Filters

Select role filter → Select status filter → Combination filtering applied

3
View Results

Table updates instantly → Shows matching staff only → Empty state if no matches

Server-Side Validation (Data Integrity)

  • Email Uniqueness: Check against existing staff, return error if duplicate
  • Employee ID Uniqueness: Unique per location (if provided)
  • Role Existence: Verify role ID exists in roles table
  • Location Access: Verify user has permission to create staff in specified location
  • Date Validation: Start date not in future, blocked dates valid ranges

Business Rules

Rule 1: Email Uniqueness

Email addresses must be unique across all staff members in the system. This ensures each staff member can be uniquely identified and prevents authentication conflicts.

Rule 2: Employee ID Uniqueness

If provided, employee IDs must be unique per location. Multi-location franchises may reuse IDs across locations, but not within the same location.

Rule 3: Start Date Validation

Start date cannot be in the future. This prevents scheduling conflicts and ensures accurate record-keeping.

Rule 4: Deactivated Staff Restrictions

Deactivated staff cannot be assigned to new jobs or shifts. Existing assignments remain for historical purposes but cannot be modified.

Rule 5: Role Assignment

Only managers can change staff roles. Role changes should trigger audit log entries and require confirmation.

Error Handling

Form Errors:
  • Display inline error messages below each field
  • Scroll to first error on submission
  • Highlight invalid fields with red border
  • Show error icon next to field label
Server Errors:
  • Display toast notification at top of page
  • Show user-friendly message (hide technical details)
  • Provide retry option for transient errors
  • Log detailed error to backend for debugging
Network Errors:
  • Show "Connection lost" message
  • Auto-retry with exponential backoff
  • Save form data to localStorage to prevent data loss
  • Offer manual retry button
